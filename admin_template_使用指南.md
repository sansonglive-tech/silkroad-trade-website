# admin_template.py 使用指南

## 一句话

这是后台管理器的**可复用模板**。以后新项目想用同样的后台，直接复制这个文件改几个参数就能用。

---

## 使用方法（新项目）

### 第 1 步：复制到新项目目录

把 `admin_template.py` 复制到你的项目文件夹里。

### 第 2 步：改配置（文件最上面）

打开文件，找到「自定义配置区」，改这几项：

```python
# 登录信息
ADMIN_USER = "admin"       # ← 改成你的用户名
ADMIN_PASS = "123456"      # ← 改成你的密码

# 文件路径 — 改成你实际的 HTML 文件名
V7_FILE = "你的网页.html"
V6_FILE = ""               # 如果你没有源文件，留空就行

# 端口
PORT = 8080

# 项目名称（显示在标题和登录页）
PROJECT_NAME = "你的项目名称"
```

往下一点还有「默认配置数据」，改你的默认内容（公司信息、服务卡片等）。

### 第 3 步：确保 HTML 文件有 CONFIG 标记

你的 HTML 文件中必须包含下面这个标记，后台才能动态更新预览：

```javascript
const CONFIG = {...}
```

如果没有的话，需要加上。

### 第 4 步：运行

```bash
python3 admin_template.py
```

然后访问：
- **后台：** `http://localhost:8080/admin`
- **预览：** `http://localhost:8080/preview`

---

## 和当前 `admin_server.py` 的关系

| 文件 | 作用 |
|------|------|
| `admin_server.py` | **当前在用的后台**（丝路山海通项目专用） |
| `admin_template.py` | **模板文件**（给以后新项目用的） |

两个互不影响，`admin_server.py` 继续正常运行。

---

## 模板 vs 当前后台的区别

- ✅ 项目名称、登录背景图都改成变量了，改 3 个地方即可
- ✅ 删掉了和国家相关的检查项（`countries-grid`），更通用
- ✅ 登录页的背景图可选，没有背景图就用纯色
- ❌ 没有动任何核心逻辑、API、前端界面

> 如果你当前项目一切正常，**不用换模板**。模板只留给以后想快速起新后台的时候用。
