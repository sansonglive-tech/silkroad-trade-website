## 📍 板块四：第二阶段——添加追踪到管理服务器

### "添加追踪"到底是什么？

> 大白话：在 JS 关键步骤加 `console.log`，让 F12 控制台能看到每一步的执行情况。出问题立刻能定位"卡在哪一步"。

**举例——页脚邮箱为什么改了没反应？**

改造前——出问题完全不知道哪一步坏了：
```javascript
document.querySelectorAll("a[href^='mailto:']").forEach(function(el){
  el.href = "mailto:" + c.company.email;
  el.textContent = c.company.email;
});
```

改造后——每一步都打印，F12 一看就知道：
```javascript
console.log('[邮箱] 找到', document.querySelectorAll("a[href^='mailto:']").length, '个链接');
document.querySelectorAll("a[href^='mailto:']").forEach(function(el, i){
  console.log('[邮箱] 第', i, '个：', el.href, '→', c.company.email);
  el.href = "mailto:" + c.company.email;
  el.textContent = c.company.email;
});
```

## 📍 板块五：第三阶段——手动发布网站

### 流程步骤

```
① 注册 GitHub（10分钟）
   - 创建仓库，选 Public
   - 勾选 Add README

② 开启 GitHub Pages（5分钟）
   - Settings → Pages
   - Source: Deploy from main / (root)

③ 本地配置 Git（10分钟）
   - 安装 Git for Windows
   - 配用户名邮箱
   - 生成 Personal Access Token（PAT，勾 repo 权限）

④ 推送代码（30分钟）
   cd C:\工作区
   git init
   git remote add origin https://...  （URL 含 PAT）
   git add . && git commit -m "first"
   git push -u origin main

⑤ 后台加发布按钮（1-2天）
   Python 一键完成：读配置 → 注入 HTML → 生成 index.html → git add → git commit → git pull --rebase → git push

⑥ 验证（10分钟）
   等待 1-2 分钟 → Ctrl+F5 刷新 → 对比本地与线上
```

### 重要提醒：分支名

> **GitHub 新仓库默认分支是 `main`，但本地 `git init` 默认是 `master`。**
>
> 如果推送到 `master`，GitHub Pages 部署的还是 `main`，所以"推送成功了但网站不更新"。
>
> 对策：统一用 `main`，或推送时 `git push origin master:main --force`

### 手动 vs 自动发布

| 场景 | 推荐方式 |
|------|---------|
| 偶尔改 | 手动：网页端上传文件 |
| 经常更新 | 自动：后台发布按钮 |
| 多人协作 | 自动 + GitHub Actions |
| 第一次 | 先手动跑通 |

## 📍 板块八：30 个常见坑汇总

| 类别 | 坑 | 解决办法 |
|------|----|---------|
| JS | 改了 href 没改 textContent | 改两遍 |
| JS | querySelector 没匹配到 | 加 console.log 看返回什么 |
| JS | script 在 DOM 之前执行 | 用 DOMContentLoaded 或 defer |
| JS | 拼接字符串缺引号 | 用模板字符串（反引号）|
| JS | 数组索引当成业务 ID | 硬编码 ID 映射数组 |
| JS | 数字拼成字符串 | 用 parseInt 转数字 |
| CSS | z-index 被遮挡 | 检查 stacking context |
| CSS | preloader 挡点击 | .hide { pointer-events: none; } |
| CSS | 字体不一致 | font-family 写在 body |
| JSON | 中文乱码 | 全程 encoding='utf-8' |
| JSON | 字段名拼错 | 定下来就别改 |
| JSON | 文件被 ANSI 保存 | 用 UTF-8 with BOM |
| HTML | 弹框 ID 不匹配 | 加 console.log 看 detailId |
| HTML | onclick 没转义 | 用反斜杠或模板字符串 |
| HTML | script 位置不对 | 移到 body 末尾 |
| Python | ANSI 写文件 | open(..., 'w', encoding='utf-8') |
| Python | 子进程卡密码 | env["GIT_TERMINAL_PROMPT"] = "0" |
| HTTP | 没点保存 | 每次都点"保存全部" |
| HTTP | 浏览器缓存 | Ctrl+F5 或换无痕窗口 |
| 网络 | github 443 连不上 | 开 VPN 或手动上传 |
| Git | push 到 master 不更新 | 查 Pages 用哪个分支 |
| Git | push 失败 fetch first | 先 git pull --rebase |
| Git | 密码认证失败 | 用 PAT 不是密码 |
| Git | 分支不统一 | 统一用 main |
| 资源 | 背景图 404 | assets/ 也要 git push |
| 资源 | 外链图片慢 | 下载到本地引用 |
| 样式 | 移动端排版乱 | 用 @media |
| 运维 | 网站崩了不知道原因 | 部署后做健康检查 |
| 隧道 | 地址每次变 | 启动后重新复制 |
| 隧道 | 本机关机就断 | 部署到云服务器 |
